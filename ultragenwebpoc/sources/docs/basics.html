UltraGen uses a inline-token syntax to setup the parts where you want to insert some data from a source. The syntax is very simple: just put a "{{" to start a piece and a "}}"

     
    <-- Simple example -->
    <div class='{{ userField }}'>
        <p>Your name: {{ name }}</p>
        <p>Ninckname: {{ lower(username) }}<p>
        <p>Birthplace: {{ @remoteBirthPlace }}<p>
    </div>

Note:

You can use another symbol but must follow the pattern "two symbols". If you want to use "<" to open and ">" to close, in the template you must write "<<" and ">>". It's particularly useful if you want to generate dynamic CSS. The symbols to open and close tokens are changed at template or at configuration file

     
    .my-class {
        background-color: << bg-color >>;
        font-family: << font-declarations >>;
    }

Printable sources

Data can be printed by three ways

  • GEN sources
  • Variables
  • Functions
GEN sources

GEN stands for Generic Enlistment Notation. It's a type of file with a simple specification with the purpose to be as simple as possible to be constructed and parsed. It consists of multiple key-value pairs linked bay a separator (default is "=", but can be changed), typically one per line. Multiple lines are accepted also. The parser run the lines and every line after a separator is considered as a multiline value from the last found key. Let's see some example, assuming the character '=' as separator:

    
        akey=a value of a key 
        anotherKey=another value to another key
        aMulti=this key will have
        a multiline value since no separator is found yet

Note:

You can change the separator in the application call. The parameters to call the application will be explained later.

Variables

Variables are a particular case of overrides, a concept that will be explained later. Beyond the data came from GEN sources, you can declare arbitrary strings to use along the template. Variables (as same as overrides) are declared using a "@" before its name. A point of attention: "@" must be the first character in the line. The syntax is as follows:

    
        @aVari:'some value'
        <p>{{ @aVari }}</p>
        # prints <p>'some value'</p>

Variables will be better explained when we talk about overrides.

Functions

UltraGen has some built-in functions to help you manipulate strings to print them the way you need. And also some funtions to print dates and files from filesystem. The syntax is:

    
        {{ upper('a lowercase will turn upper') }}
        #prints A LOWERCASE WILL TURN UPPER

Let's mix everything?


            aName=It's a gen source
            description=This is my very first UltraGen try
        

        <!-- the template below... -->
        @title:upper(aName)
        <h2>{{ @title }}</h2>
        <p>{{ description }}</p>
        <small>posted on {{ date(NOW) }}</small>
    

        <!-- ...brings the follows -->
        <h2>IT'S A GEN SOURCE</h2>
        <p>This is my very first UltraGen try</p>
        <small>posted on 2019-10-26 13:06:20.223</small>
